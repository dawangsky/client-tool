# 模块 9：单据/消息

## 目标

关键协议/合同留痕、交易确认与失败通知。

## API（V1）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/v1/contracts/{type}` | 获取协议模板（适当性揭示、基金合同摘要等） |
| POST | `/api/v1/contracts/sign` | 签署留痕（关联 `suitability_log` 或 `order_no`） |
| GET | `/api/v1/confirm-docs/{orderNo}` | 交易确认单 |
| GET | `/api/v1/messages` | 消息列表（站内） |

## 业务规则

- 适当性揭示、高风险产品确认必须在下单前完成并留痕
- 确认成功/失败、退款完成需触发消息（站内 + 可扩展短信）

## 验收

- [ ] 下单可追溯已签署协议
- [ ] 确认/失败客户可查到通知记录
