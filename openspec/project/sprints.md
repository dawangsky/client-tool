# V1 迭代顺序（Sprint）

与需求文档 §8 对齐；每个 Sprint 结束应有可演示增量。

## Sprint 1 — 账户 + 产品 + 适当性 + 交易骨架

**目标**：C 端可注册开户、看产品、完成风测、创建「受理中」订单（不含真实扣款/报 TA）。

| 端 | 交付 |
|----|------|
| 后端 | 客户/交易账户/绑卡表；产品/净值查询；风测与适当性留痕；`trade_order` 创建与状态机骨架 |
| 前端 | 登录注册、开户流程、产品列表/详情、风测问卷、申购/赎回下单页（仅受理） |
| 契约 | `account` `product` `suitability` `trade` 模块 API v0 |

## Sprint 2 — 支付扣款/退款 + 资金流水

**目标**：申购扣款成功/失败；`pay_order` + `capital_flow`；失败可退款入口。

| 端 | 交付 |
|----|------|
| 后端 | 支付网关适配层；扣款/退款；与 `order_no` 关联 |
| 前端 | 支付结果页、订单支付状态展示 |
| 契约 | `payment` 模块 API |

## Sprint 3 — 清算报 TA + 确认 + 持仓

**目标**：报盘、收确认、持仓加仓。

| 端 | 交付 |
|----|------|
| 后端 | `clear_batch`/`clear_detail`/`confirm_result`；持仓过账 |
| 前端 | 持仓列表、交易明细、确认结果展示 |
| 契约 | `clearing` `position` 模块 API |

## Sprint 4 — 赎回全链路 + 出款

**目标**：冻结份额 → 报 TA → 确认 → 出款到卡 → 减仓。

## Sprint 5 — 对账 + 运营后台 + 单据消息 + 验收

**目标**：日终对账任务、差错处理入口、协议/通知、全链路验收清单通过。
