# V1 范围说明

> 来源：`基金代销系统-V1最小功能模块.md`  
> 口径：公募场外代销；先打通「一人一平台、一只场外基金」闭环。

## 1. 第一版定位

| 项 | 说明 |
|----|------|
| 业务闭环 | 开户 → 申购扣款 → 报 TA → 确认份额 → 持仓 → 赎回出款 |
| 合规底线 | 适当性匹配、揭示留痕、资金与自有资金隔离意识 |
| 工程底线 | 订单状态机、支付/TA 对账、失败退款 |

**不做完对账与退款，不算 V1 可上线。**

## 2. Must（V1 必须）

- 开户 / 登记（交易账户 + 代向 TA 申请基金账户）
- 申购、赎回
- 截点前撤单
- 确认成功 / 失败（失败自动退款）
- 持仓查询、交易查询
- 10 大模块最小能力（见根 README 映射表）

## 3. Won't（V1 不做）

| 能力 | 目标版本 |
|------|----------|
| 认购（募集期） | V1.1 |
| 转换 / 超级转换 | V2 |
| 转托管 | V2 |
| 定投 | V2 |
| 货币 T+0 快赎 | V2+ |
| B 端 OpenAPI | V2+ |
| 分润 / 复杂营销 | V2+ |

## 4. 账户口径（实现必守）

- **交易账户**：销售公司开通（`trade_account`）
- **基金账户**：TA 开通，销售代申请；同一人同一 TA 复用，不同 TA 新建
- **支付成功 ≠ 申购成功**：份额以 TA 确认为权威

## 5. 责任分界

| 域 | 主责 |
|----|------|
| 支付清结算 | 扣款/出款/通道对账/退款执行 |
| 基金清结算 | 报 TA、确认份额费用、持仓过账 |

- **申购**：支付先、基金后  
- **赎回**：基金先、支付后  

## 6. 上线验收清单

- [ ] 新客：开户 → 扣款申购 → 确认 → 持仓正确
- [ ] 赎回：冻份额 → 确认 → 出款到卡
- [ ] 确认失败：自动退款且订单终态正确
- [ ] 日终：通道账、TA 确认、本地订单/持仓可平
- [ ] 适当性不匹配可阻断并留痕
- [ ] 截点前撤单可用；截点后不可撤
- [ ] `order_no` 可串联交易/支付/清算
- [ ] 监控：已扣款未报盘、确认失败未退款
