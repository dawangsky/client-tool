# 桌面客户端 UI 规范（fund-mac）

面向 **macOS 基金代销客户端**，观感对齐市面正规财富管理 / 基金销售 App（稳重、可信任），而非营销落地页或「AI 模板页」。

## 原则

1. **品牌与信任感优先**：深蓝 / 墨绿 / 炭灰为主，克制青绿强调色；避免紫粉渐变、霓虹、大面积毛玻璃。
2. **信息架构像客户端**：侧栏导航 + 主内容；首屏是资产概览 / 热门产品 / 交易入口，不是 Landing Hero。
3. **列表与表单密而不乱**：表格对齐、等宽数字、清晰字阶；少卡片墙、少徽章贴纸。
4. **行情惯例**：红涨绿跌（`--quote-up` / `--quote-down`）。
5. **动效克制**：仅页面过渡与 hover，勿炫技。

## 主题变量

见 `src/styles/tokens.css`：

| Token | 用途 |
|-------|------|
| `--brand-navy` | 品牌主色 / 侧栏 |
| `--brand-accent` | 主按钮 / 选中态 |
| `--brand-gold` | 侧栏选中指示 |
| `--surface-*` | 页面 / 面板背景 |
| `--quote-up/down` | 涨跌色 |

字体：`Source Serif 4` + `Noto Serif SC`（标题），`IBM Plex Sans` + `Noto Sans SC`（正文），`IBM Plex Mono`（数字/代码）。

## 布局

- 左侧固定侧栏（约 220px）：总览 / 交易 / 账户
- 顶栏：当前页标题 + Mock/联调标识 + 时钟
- 主区：可滚动内容，最大宽度按表格友好排版

## 与 Web 差异

| 项 | Web (`fund-front`) | Mac (`fund-mac`) |
|----|--------------------|------------------|
| 壳 | 顶栏横向导航 | 侧栏桌面壳 |
| 路由 | `createWebHistory` | `createWebHashHistory`（适配 Tauri） |
| 设备 | 无 | `X-Device-Id` / `X-Device-Type` + Rust 持久化 deviceId |
| 审计 | 无客户端队列 | `src/api/audit.ts` 本地队列 + 上报预留 |
| 运营后台 | 有占位 | **不做**（C 端桌面） |

## 验收对照

- [ ] 侧栏可到达：登录、产品、适当性、申赎、订单、持仓、消息
- [ ] 视觉无紫粉霓虹 / 假仪表盘堆砌
- [ ] CSS 变量可一处换肤
